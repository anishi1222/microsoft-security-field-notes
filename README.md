# microsoft-security-field-notes

このリポジトリは、検証した Microsoft Security 関連アプリケーションやソリューションを保存し、内容を整理して蓄積していくためのフィールドノート用リポジトリです。

現在は Vuln-Notification を格納しています。今後も検証済みのアプリケーションを追加し、それぞれの用途や構成、検証結果を記録していく想定です。

## このリポジトリについて

- 想定読者: Microsoft Security 関連ソリューションを評価・再利用したい利用者
- 目的: 検証済みソリューションの概要、構成、検証結果を再利用しやすい形で残すこと
- 役割分担:
  - ルート README: 全体方針、使い方、収録一覧
  - 各ソリューション配下の README: 実装・構築・運用の詳細

## 活用方針

公開リポジトリとして、次の方針で運用します。

1. 追加するソリューションの記載ルール
	- 目的、想定ユースケース、構成要素を最初に明示する
	- 実行手順は再現可能な最小単位で記載する
2. 検証結果の残し方
	- 検証日、前提条件、確認できたこと、未確認事項を分けて記載する
	- 制約や既知課題は必ず明記する
3. サンプル値と秘匿情報の扱い
	- テナント ID、ユーザー情報、キー、実環境名は掲載しない
	- 公開可能なサンプル値またはプレースホルダーを使用する
4. 更新・保守の方針
	- 大きな仕様変更時は README と実装を同時に更新する
	- 非推奨化した内容は README に追記して利用者に明示する

## 使い方

1. ルート README で全体像を把握する
2. 目的に合うソリューションを 収録ソリューション から選ぶ
3. 選んだソリューション配下の README を参照する
4. 必要に応じて RUNBOOK やテストスクリプトで実行・検証する

## 収録ソリューション

### Vuln-Notification

Azure Functions と Microsoft Graph を使って、脆弱性情報の通知とタスク化を行うサンプルです。

- 概要・構成・手順: [vuln-notification/README.md](vuln-notification/README.md)

## 新規ソリューション追加時の最低要件

新しいディレクトリを追加する場合は、原則として以下を揃えます。

- README: 目的、前提条件、構成、実行手順
- 検証結果: 何を確認し、何が未確認か
- 公開配慮: 秘匿情報を除去し、サンプル値へ置換
